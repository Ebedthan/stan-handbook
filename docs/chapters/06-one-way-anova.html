6  One-Way ANOVA in Stan

6.1 Overview

One-way ANOVA is one of the most frequently used tools in life-science research: does the outcome differ across treatment groups? Students learn ANOVA as an \(F\)-test delivered by aov(). This chapter shows that ANOVA is not a separate procedure, it is linear regression with a categorical predictor, and it lives in Stan as naturally as the continuous-predictor models of Chapters 4 and 5.

The chapter introduces two parameterisations of the same model, treatment coding and sum-to-zero coding, explains why both exist, and shows how to translate Stan posterior output back into the quantities an ANOVA table would report. Along the way, we gain tools that classical ANOVA cannot provide: posterior probabilities for pairwise contrasts, honest uncertainty about group means, and a principled way to handle unbalanced designs.

Learning objectives:

By the end of this chapter you will be able to:

  • Explain why one-way ANOVA is a special case of linear regression and write both as a Stan model.
  • Choose between treatment coding and sum-to-zero coding, implement each in Stan, and interpret the parameters of each.
  • Extract posterior group means, pairwise differences, and their credible intervals directly from Stan draws.
  • Reconstruct the quantities reported in a classical ANOVA table (\(F\)-statistic, \(\eta^2\)) from the posterior, and explain their limitations.
  • Perform a posterior predictive check appropriate for grouped data.

6.2 ANOVA is regression

Consider a completely randomised experiment with \(K\) groups. The classical ANOVA model is:

\[ y_{ij} = \mu + \tau_k + \varepsilon_{ij}, \qquad \varepsilon_{ij} \sim \text{Normal}(0, \sigma) \]

where \(\mu\) is the grand mean, \(\tau_k\) is the effect of group \(k\), and the constraint \(\sum_k \tau_k = 0\) makes the model identifiable.

Now write this as a regression. Create \(K - 1\) binary dummy variables \(d_{ik}\): \(d_{ik} = 1\) if observation \(i\) belongs to group \(k\), zero otherwise. Then:

\[ y_i = \alpha + \beta_2 d_{i2} + \beta_3 d_{i3} + \cdots + \beta_K d_{iK} + \varepsilon_i \]

Here \(\alpha\) is the mean of group 1 (the reference), and \(\beta_k\) is the difference between group \(k\) and group 1. This is exactly what lm(y ~ factor(group)) does in R, the ANOVA and the regression are the same model, just parameterised differently.

In Stan, we can use either parameterisation. The choice affects how we interpret the parameters but not the predictions or the fit.

6.3 Running example: soil carbon under land use types

We model topsoil organic carbon (SOC, g kg⁻¹) across four land use types common in the West African forest-savanna mosaic: natural forest, agroforest, fallow, and cropland. The scientific question is whether land use type is associated with different SOC levels, a one-way ANOVA with \(K = 4\) groups.

Code
set.seed(2025)

# True group means (g/kg SOC, broadly consistent with West African
# forest-savanna literature)
K <- 4
group_names <- c("Natural forest", "Agroforest", "Fallow", "Cropland")
true_mu <- c(18.5, 14.2, 11.8, 8.4)   # group means
true_sigma <- 2.8                     # common within-group SD

# Unbalanced design (realistic: fewer forest plots)
n_per_group <- c(10, 14, 14, 12)
N <- sum(n_per_group)

group <- rep(seq_len(K), times = n_per_group)
y <- rnorm(N, mean = true_mu[group], sd = true_sigma)

soc_df <- data.frame(
  y = y,
  group = group,
  group_name = factor(group_names[group], levels = group_names)
)

# Group means for plot
group_means <- tapply(y, soc_df$group_name, mean)
gm_df <- data.frame(
  group_name = names(group_means),
  mean = as.numeric(group_means)
)

ggplot(soc_df, aes(group_name, y)) +
  geom_jitter(width = 0.12, colour = "#2e7d6a", alpha = 0.7, size = 2) +
  geom_crossbar(data = gm_df,
                aes(y = mean, ymin = mean, ymax = mean),
                colour = "#1a4a40", linewidth = 0.9, width = 0.45) +
  labs(
    x = "Land use type",
    y = "Topsoil SOC (g kg⁻¹)",
    title = "Soil organic carbon by land use type"
  )

Why an unbalanced design?
Natural forest plots are often scarcer and harder to access than cropland plots. With \(n = (10, 14, 14, 12)\), this is a mildly unbalanced design. Classical ANOVA handles unbalanced designs awkwardly (Type I vs Type III sums of squares). Bayesian ANOVA handles them transparently, the likelihood simply reflects the different amounts of information in each group.

6.4 Parameterisation 1: treatment coding

6.4.1 The Stan model

data {
  int<lower=1> N;
  int<lower=2> K;
  array[N] int<lower=1, upper=K> group;
  vector[N] y;
}

parameters {
  real alpha;         // reference group mean (group 1)
  vector[K-1] tau;    // deviations from reference (groups 2..K)
  real<lower=0> sigma;
}

transformed parameters {
  vector[K] mu;
  mu[1] = alpha;
  for (k in 2:K)
    mu[k] = alpha + tau[k-1];
}

model {
  alpha ~ normal(0, 10);
  tau ~ normal(0, 5);
  sigma ~ exponential(1);

  for (i in 1:N)
    y[i] ~ normal(mu[group[i]], sigma);
}

generated quantities {
  vector[N] y_rep;
  vector[N] log_lik;

  for (i in 1:N) {
    y_rep[i] = normal_rng(mu[group[i]], sigma);
    log_lik[i] = normal_lpdf(y[i] | mu[group[i]], sigma);
  }
}

6.4.2 Key design decisions

array[N] int group is Stan’s syntax for an integer array. One group index per observation. Stan 2.26+ uses the array[N] int declaration; older syntax was int group[N]. Always use the modern form.

transformed parameters block. We assemble the full group mean vector mu here rather than in the model block. This means mu is saved in the posterior draws automatically, we can inspect each group mean without any post-processing arithmetic.

vector[K-1] tau declares \(K - 1\) free treatment effects. Group 1 is the reference, so tau[1] is the group 2 vs group 1 difference, tau[2] is group 3 vs group 1, and so on. This is exactly what lm(y ~ factor(group)) returns.

6.4.3 Compile and sample

mod_tc <- cmdstan_model("stan/anova_treatment.stan")

}

6.5 Parameterisation 2: sum-to-zero coding

6.5.1 Why sum-to-zero?

Treatment coding makes group 1 special, all other effects are relative to it. The choice of reference group affects the values (and priors) of every tau parameter, which can be awkward when no group is a natural reference.

Sum-to-zero coding centres all effects around the grand mean \(\mu\):

\[ y_i \sim \text{Normal}(\mu + \tau_{g(i)},\; \sigma), \qquad \sum_{k=1}^K \tau_k = 0 \]

Now \(\mu\) is the grand (unweighted) mean and \(\tau_k\) is the deviation of group \(k\) from it. The constraint \(\sum \tau_k = 0\) is imposed by deriving the last effect: \(\tau_K = -\sum_{k=1}^{K-1} \tau_k\).

6.5.2 The Stan model

data {
  int<lower=1> N;
  int<lower=2> K;
  array[N] int<lower=1, upper=K> group;
  vector[N] y;
}

parameters {
  real mu;              // grand mean
  vector[K-1] tau_raw;  // free effects for groups 1..K-1
  real<lower=0> sigma;
}

transformed parameters {
  vector[K] tau;
  tau[1:(K-1)] = tau_raw;
  tau[K] = -sum(tau_raw);   // sum-to-zero constraint
}

model {
  mu ~ normal(0, 10);
  tau_raw ~ normal(0, 5);
  sigma ~ exponential(1);

  for (i in 1:N)
    y[i] ~ normal(mu + tau[group[i]], sigma);
}

generated quantities {
  vector[N] y_rep;
  vector[N] log_lik;

  for (i in 1:N) {
    y_rep[i] = normal_rng(mu + tau[group[i]], sigma);
    log_lik[i] = normal_lpdf(y[i] | mu + tau[group[i]], sigma);
  }
}

Implementing the sum-to-zero constraint.
Stan cannot impose sum(tau) == 0 as a hard constraint in the parameters block — that would reduce the parameter space dimension. Instead, we declare K-1 free parameters (tau_raw) and derive the \(K\)-th in transformed parameters as tau[K] = -sum(tau_raw). Stan samples freely in the \((K-1)\)-dimensional unconstrained space and the constraint holds exactly for every draw. This is the standard Stan idiom for sum-to-zero constraints.

Both models give identical group means.
Treatment coding: group mean \(k\) = alpha + tau[k-1]. Sum-to-zero coding: group mean \(k\) = mu + tau[k]. The two parameterisations are mathematically equivalent, they differ only in what the named parameters represent, not in what the model predicts. Verify this by comparing mu[k] from the treatment model with mu + tau[k] from the sum-to-zero model: they should match to numerical precision.

6.6.2 Pairwise contrasts

Classical ANOVA follows up a significant \(F\)-test with post-hoc tests (Tukey HSD, Bonferroni). In Stan, pairwise contrasts are just arithmetic on the posterior draws, no adjustment for multiple comparisons is needed because we are estimating probabilities, not controlling Type I error rates.

Code
draws_mu <- as_draws_df(fit_tc$draws(c("mu[1]", "mu[2]",
                                        "mu[3]", "mu[4]")))
names(draws_mu)[1:4] <- group_names

# All pairwise differences
pairs <- combn(group_names, 2, simplify = FALSE)

contrast_df <- do.call(rbind, lapply(pairs, function(p) {
  diff <- draws_mu[[p[1]]] - draws_mu[[p[2]]]
  data.frame(
    contrast = paste(p[1], "-", p[2]),
    mean = round(mean(diff), 2),
    lo90 = round(quantile(diff, 0.05), 2),
    hi90 = round(quantile(diff, 0.95), 2),
    p_positive = round(mean(diff > 0), 3)
  )
}))

6.7 Connecting to the classical ANOVA table

Students expect to see an ANOVA table with \(SS\), \(df\), \(MS\), and the \(F\)-statistic. We can reconstruct these from the posterior, but more importantly we can explain what they actually measure, and why the Bayesian alternatives are more useful.

6.7.2 Posterior \(R^2\) and \(\eta^2\)

The classical effect size \(\eta^2 = SS_{\text{between}} / SS_{\text{total}}\) measures the proportion of variance explained by group membership. We can compute a Bayesian analogue directly from the posterior.

Code
# Bayesian R²
# R² = Var(fitted) / (Var(fitted) + sigma²)
draws_all <- as_draws_df(
  fit_tc$draws(c("mu[1]", "mu[2]", "mu[3]", "mu[4]", "sigma"))
)

# Column names are "mu[1]"..."mu[4]" and "sigma", use them directly
mu_cols <- paste0("mu[", 1:K, "]")

bayes_r2 <- sapply(seq_len(nrow(draws_all)), function(i) {
  # Map each observation to its group mean draw
  mu_obs <- as.numeric(draws_all[i, mu_cols])[group]
  var_fit <- var(mu_obs)
  sigma_i <- draws_all$sigma[i]
  var_fit / (var_fit + sigma_i^2)
})

cat("Posterior mean Bayesian R²: ", round(mean(bayes_r2), 3), "\n")
#> Posterior mean Bayesian R²:  0.609
Code
cat("90% credible interval: [",
    round(quantile(bayes_r2, 0.05), 3), ",",
    round(quantile(bayes_r2, 0.95), 3), "]\n")
#> 90% credible interval: [ 0.477 , 0.717 ]
Code
cat("Classical eta²:             ",
    round(summary(aov_fit)[[1]]["factor(group)", "Sum Sq"] /
          sum(summary(aov_fit)[[1]][, "Sum Sq"]), 3), "\n")
#> Classical eta²:              0.668

Why Bayesian \(R^2\) instead of \(\eta^2\)?
Classical \(\eta^2\) is a single number computed from point estimates. Bayesian \(R^2\) (gelman2019r2?) is a full posterior distribution: it tells you not just the most likely proportion of variance explained, but how uncertain that proportion is. With small samples, the credible interval for \(R^2\) can be very wide, a fact \(\eta^2\) conceals entirely.

6.7.3 Reconstructing the \(F\)-statistic

The \(F\)-statistic can be recovered from the posterior mean estimates, but this is rarely useful that is the posterior gives richer information directly. We show it here purely for comparison.

Code
# Posterior mean group means
mu_hat  <- colMeans(draws_all[, mu_cols])
sigma_hat <- mean(draws_all$sigma)

# Between-group variance (weighted by n)
grand_hat <- weighted.mean(mu_hat, n_per_group)
ss_between <- sum(n_per_group * (mu_hat - grand_hat)^2)
ms_between <- ss_between / (K - 1)
ms_within  <- sigma_hat^2

cat("F from posterior means:  ", round(ms_between / ms_within, 2), "\n")
#> F from posterior means:   25.06
Code
cat("F from classical ANOVA:  ",
    round(summary(aov_fit)[[1]]["factor(group)", "F value"], 2), "\n")
#> F from classical ANOVA:   30.8

6.10 Summary

  • One-way ANOVA is linear regression with a categorical predictor. In Stan, the two natural parameterisations are treatment coding (effects relative to a reference group) and sum-to-zero coding (effects relative to the grand mean). Both give identical group means and fits.
  • Treatment coding uses real alpha and vector[K-1] tau; the full group mean vector is assembled in transformed parameters. Sum-to-zero coding uses real mu and vector[K-1] tau_raw, deriving the last effect as tau[K] = -sum(tau_raw).
  • Pairwise contrasts are arithmetic on posterior draws, no multiplicity adjustment is required, and the result is a full posterior distribution for each difference, not just a \(p\)-value.
  • Bayesian \(R^2\) generalises \(\eta^2\) to a posterior distribution that honestly represents uncertainty about the proportion of variance explained.
  • Unbalanced designs are handled transparently by the likelihood; no special treatment is required.
  • The ppc_dens_overlay_grouped() function from bayesplot provides a group-specific predictive check, more informative than the marginal overlay used in earlier chapters.

6.11 Exercises

  1. Change the reference group. Refit the treatment-coded model setting cropland (group 4) as the reference. How do alpha and the tau values change? Do the group means mu[k] change? Why or why not?

  2. Verify the equivalence. Using posterior draws from both the treatment-coded and sum-to-zero models, compute the group means implied by each (alpha + tau[k-1] vs mu + tau[k]). Show that they match to numerical precision using all.equal().

  3. Ordered contrasts. Ecologists often expect a monotone gradient: forest > agroforest > fallow > cropland. Using the pairwise contrast draws, compute the posterior probability that all three adjacent pairs are in the expected direction simultaneously:

    mean(
  (draws_mu[["Natural forest"]] > draws_mu[["Agroforest"]]) &
  (draws_mu[["Agroforest"]] > draws_mu[["Fallow"]]) &
  (draws_mu[["Fallow"]] > draws_mu[["Cropland"]])
)

    What is this probability? Is it larger or smaller than you expected?

  4. Heteroscedastic ANOVA. The homoscedastic assumption (common \(\sigma\) across groups) can be relaxed. Modify the treatment-coded Stan model to estimate a separate sigma[k] for each group: replace real<lower=0> sigma with vector<lower=0>[K] sigma and update the likelihood to normal(mu[group[i]], sigma[group[i]]). Refit and compare LOO scores. Is the heteroscedastic model preferred?

  5. Sample size sensitivity. Rerun the simulation with n_per_group <- c(3, 4, 4, 3) (very small groups). How do the posterior group means compare to the OLS means? Where does the Bayesian approach benefit most from the prior?
